## 👤 Роль: Сотрудник

### US-001: Заявление на отгул

**Статус:** ✅ Готово

**Как** сотрудник,  
**я хочу** отметить день отгула в календаре,  
**чтобы** он автоматически учитывался в графике отсутствия

**✅ Критерии приёмки:**

- [x] В календаре есть кнопка «Добавить отгул»
- [x] При выборе типа «Отгул» сохраняется дата и комментарий (опционально)
- [x] Отгул корректно отображается в личном календаре
- [x] Сотрудник может скорректировать заявку до начала отгула
- [x] Сотрудник может отменить заявку до начала отгула

**📝 Заметки / Ограничения:**
- Подтверждение руководителем не требуется
- Валидация дат (например, запрет отгулов в праздники) будет добавлена позднее

### US-003: Автоматическое уведомление

**Статус:** 📋 Бэклог

**Как** сотрудник,  
**я хочу** чтобы при создании отгула или больничного в корпоративный мессенджер автоматически отправлялось уведомление руководителю и HR,  
**чтобы** не вспоминать, кому нужно писать, и не дублировать информацию вручную

**✅ Критерии приёмки:**
- [ ] При сохранении заявки автоматически формируется сообщение в корпоративный мессенджер
- [ ] В уведомлении указаны: ФИО сотрудника, тип отсутствия, период
- [ ] Уведомление уходит только руководителю отдела и HR
- [ ] При ошибке отправки статус фиксируется в системных логах, интерфейс пользователя не блокируется

**📝 Заметки / Ограничения:**
- Реализуются только исходящие уведомления. Ответы из мессенджера не обрабатываются.

## 👤 Роль: HR

### US-004: Автоматическое напоминание сотрудникам

**Статус:** 📋 Бэклог

**Как** HR,  
**я хочу** автоматическое напоминание о необходимости заполнения календаря отгулов и больничных,  
**чтобы** не писать каждый месяц в чаты каждого отдела вручную

**✅ Критерии приёмки:**
- [ ] В заданное время (по умолчанию последний день месяца) бот отправляет сообщение в чат отдела
- [ ] Сообщение содержит краткую инструкцию и прямую ссылку на календарь
- [ ] Рассылка не дублируется, если уже была выполнена за текущий месяц
- [ ] Ошибка отправки логируется без прерывания работы системы

**📝 Заметки / Ограничения:**
- Административная страница для настройки времени/даты/текста сообщения будет добавлена позднее
- Умный расчёт «последнего рабочего дня» с учётом праздников и выходных будет добавлена позднее

---

### US-005: Выгрузка сводных данных об отсутствиях

**Статус:** 📋 Бэклог

**Как** HR,  
**я хочу** выгружать единый файл с данными об отгулах и больничных сотрудников,  
**чтобы** не создавать сводные таблицы вручную

**✅ Критерии приёмки:**
- [ ] На странице доступна кнопка «Выгрузить отчёты»
- [ ] Формируется файл (CSV) со столбцами: ФИО, отдел, тип отсутствия, даты, статус
- [ ] Выгрузка учитывает все активные записи за выбранный период (по умолчанию текущий месяц)
- [ ] При отсутствии данных за период генерируется файл с заголовками и пустыми строками

**📝 Заметки / Ограничения:**
- Фильтры по отделам/сотрудникам и гибкий выбор периода будут добавлены позднее
- Выгрузка доступна только пользователям с ролью HR/Admin
